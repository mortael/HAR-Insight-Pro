# Building HAR Insight Pro Desktop App

This guide provides detailed instructions for building and distributing the HAR Insight Pro Windows desktop application.

## Prerequisites

Before you begin, ensure you have the following installed:
- **Node.js**: v16 or higher (v18 LTS recommended)
- **npm**: Comes with Node.js
- **Git**: For version control

## Development Setup

### 1. Clone the Repository

```bash
git clone https://github.com/mortael/HAR-Insight-Pro.git
cd HAR-Insight-Pro
```

### 2. Install Dependencies

```bash
npm install
```

This will:
- Install all Node.js dependencies
- Automatically generate application icons from the SVG source
- Set up Electron and electron-builder

### 3. Run in Development Mode

#### As a Web Application

```bash
npm run dev
```

The application will be available at http://localhost:3000

#### As a Desktop Application

```bash
npm run electron:dev
```

This starts both:
- Vite development server on port 3000
- Electron window that loads the dev server
- Hot Module Replacement (HMR) for instant updates

## Building for Production

### Web Build Only

To build just the web assets:

```bash
npm run build
```

Output: `dist/` directory

### Windows Desktop App

To build the Windows desktop application:

```bash
npm run electron:build:win
```

This will:
1. Build the web application using Vite
2. Package the app with Electron
3. Create Windows installers

Output: `release/` directory containing:
- `HAR-Insight-Pro-1.0.0-x64.exe` - NSIS installer
- `HAR-Insight-Pro-1.0.0-portable.exe` - Portable executable

### Build for All Platforms

```bash
npm run electron:build
```

Note: Building for macOS requires running on macOS, and building for Linux works on any platform.

## Icon Management

### Icon Files

The application uses a custom-designed logo located in `build/icon.svg`. During `npm install`, the following icon files are automatically generated:

- `icon.png` - Main 512x512 icon
- `icon_16x16.png` through `icon_1024x1024.png` - Various sizes
- Windows ICO file is generated by electron-builder during packaging

### Regenerating Icons

If you modify the SVG logo:

```bash
node scripts/generate-icons.cjs
```

### Customizing the Logo

1. Edit `build/icon.svg` with your design
2. Keep the viewBox at `0 0 512 512` for best results
3. Run the icon generation script
4. Test the icons in development mode

## Configuration

### Electron Builder (electron-builder.json)

Key configurations:
- **appId**: `com.harinsightpro.app`
- **productName**: `HAR Insight Pro`
- **Windows targets**: NSIS installer + Portable
- **Architecture**: x64

To add more build targets, modify `electron-builder.json`:

```json
{
  "win": {
    "target": [
      {"target": "nsis", "arch": ["x64"]},
      {"target": "portable", "arch": ["x64"]},
      {"target": "zip", "arch": ["x64"]}
    ]
  }
}
```

### Vite Configuration (vite.config.ts)

Key settings for Electron:
- `base: './'` - Relative paths for Electron
- `build.outDir: 'dist'` - Output directory

### Package.json Scripts

- `npm run dev` - Web dev server
- `npm run build` - Web production build
- `npm run electron:dev` - Desktop dev mode
- `npm run electron:build` - Build desktop app
- `npm run electron:build:win` - Build Windows only

## Troubleshooting

### Issue: Electron window is blank

**Solution**: Ensure the build completed successfully and the `dist/` directory exists.

```bash
npm run build
```

### Issue: Icons not showing

**Solution**: Regenerate icons:

```bash
node scripts/generate-icons.cjs
npm run build
```

### Issue: Build fails on Windows

**Solution**:
1. Ensure you have Windows Build Tools installed
2. Run as Administrator if needed
3. Check antivirus isn't blocking electron-builder

### Issue: "Module not found" errors

**Solution**: Clean install:

```bash
rm -rf node_modules package-lock.json
npm install
```

## Distribution

### Creating a Release

1. Update version in `package.json`
2. Build the application:
   ```bash
   npm run electron:build:win
   ```
3. Test the installer in `release/`
4. Create a GitHub release
5. Upload the installers as release assets

### File Sizes

Typical sizes:
- Installer: ~80-100 MB
- Portable: ~150-170 MB (includes everything)
- Web build: ~400 KB (without node_modules)

### System Requirements

Minimum requirements for Windows:
- Windows 7 or later
- 64-bit processor
- 100 MB free disk space
- 512 MB RAM (2 GB recommended)

## Security Considerations

The application follows Electron security best practices:

1. **Context Isolation**: Enabled in `electron/main.js`
2. **Node Integration**: Disabled
3. **Preload Script**: Used for safe IPC
4. **CSP**: Consider adding Content Security Policy headers
5. **Auto-updates**: Can be added with electron-updater

### Adding Auto-Updates

1. Install electron-updater:
   ```bash
   npm install electron-updater
   ```

2. Modify `electron/main.js` to include update checking

3. Configure publishing in `electron-builder.json`

## Advanced Topics

### Code Signing (Recommended for Distribution)

For production releases, code signing is recommended:

1. Obtain a code signing certificate
2. Configure in `electron-builder.json`:
   ```json
   {
     "win": {
       "certificateFile": "path/to/cert.pfx",
       "certificatePassword": "your-password"
     }
   }
   ```

### Custom Installer

Modify NSIS installer options in `electron-builder.json`:

```json
{
  "nsis": {
    "oneClick": false,
    "allowToChangeInstallationDirectory": true,
    "createDesktopShortcut": true,
    "createStartMenuShortcut": true,
    "installerIcon": "build/icon.ico",
    "uninstallerIcon": "build/icon.ico",
    "license": "LICENSE.txt"
  }
}
```

## Performance Optimization

### Reducing Bundle Size

1. Use production builds
2. Enable tree shaking (already configured in Vite)
3. Analyze bundle with:
   ```bash
   npm run build -- --mode analyze
   ```

### Faster Development

1. Use `npm run electron:dev` for hot reload
2. Disable dev tools in production
3. Use React DevTools extension

## CI/CD Integration

### GitHub Actions Example

```yaml
name: Build Desktop App

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npm run electron:build:win
      - uses: actions/upload-artifact@v3
        with:
          name: windows-installers
          path: release/*.exe
```
